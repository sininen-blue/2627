What is translation in the context of language levels?

- Converting each L1 instruction into equivalent L0 instructions to produce a standalone program
- Running an L1 program directly on the hardware without conversion
- Interpreting L1 instructions one at a time during execution
- Converting binary signals into symbolic assembly code

---

What does an interpreter do?

- Reads L1 instructions and translates them on-the-fly for the hardware to execute
- Compiles the entire L1 program into an L0 binary before execution
- Converts symbolic assembly language into numeric machine code
- Manages memory allocation and concurrency at the operating system level

---

What is the Digital Logic level composed of?

- Gates built from transistors that process binary signals
- Registers and an arithmetic logic unit connected by a data path
- Symbolic instructions that represent lower-level operations
- High-level language constructs like variables and functions

---

What does the ALU do in the microarchitecture level?

- Operates on data stored in registers to perform arithmetic and logical operations
- Translates assembly language into machine code
- Fetches and decodes instructions from memory
- Manages input and output between peripherals and the CPU

---

What is the difference between architecture and implementation?

- Architecture is visible to the user; implementation is how features are realized
- Architecture is software; implementation is hardware
- Architecture refers to gates; implementation refers to languages
- There is no difference between the two

---

At which boundary does a shift from numeric to symbolic representations occur?

- Between the Operating System level and the Assembly Language level
- Between the Digital Logic level and the Microarchitecture level
- Between the Microarchitecture level and the ISA level
- Between the Assembly Language level and the Problem-Oriented Language level

---

What does an assembler do?

- Converts symbolic assembly language into numeric machine code
- Translates high-level code into assembly language
- Interprets programs directly on the hardware
- Handles memory management and process scheduling

---

What does it mean when we say L1 runs on M0 in the virtual machine model?

- The M0 machine interprets or executes programs written in L1 after translation
- L1 is physically built into the M0 hardware
- M0 and L1 refer to the same language
- L1 bypasses M0 and runs directly on the CPU

---

What is the main purpose of abstraction layers in computer design?

- Each layer suppresses irrelevant detail to reduce complexity
- Each layer adds more hardware to increase processing speed
- Each layer completely replaces the one below it
- Each layer eliminates the need for lower-level languages

---

What is the correct level ordering from highest to lowest in a modern computer?

- Problem-Oriented Language → Assembly Language → Operating System → ISA → Microarchitecture → Digital Logic
- Digital Logic → Microarchitecture → ISA → Operating System → Assembly Language → Problem-Oriented Language
- Assembly Language → Problem-Oriented Language → ISA → Operating System → Digital Logic → Microarchitecture
- Operating System → Digital Logic → Microarchitecture → ISA → Assembly Language → Problem-Oriented Language
